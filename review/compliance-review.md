# 要件反映レビュー結果

## 総合評価
要修正

## 指摘事項

### 🔴 Critical（必須対応）

- **[トレーサビリティ欠如] ヒアリング・ビジネス課題との対応付けが存在しない**
  各要件がどのビジネス課題・ヒアリング発言から導出されたか一切追跡できない。要件変更時の影響分析や合意根拠の確認が不可能な状態。ヒアリング議事録 ID または課題管理 ID を要件・機能一覧の列として追加すること。

- **[機能詳細の欠落] F-003（地図表示）の機能詳細が未記載**
  全ユーザーストーリー（US-001〜US-009）に関連する最重要機能にもかかわらず、「概要・入力・出力・備考」のセクションが存在しない。ズーム範囲・初期表示位置・タイル配信元 URL 等の仕様を明記すること。

- **[機能詳細の欠落] F-005・F-007〜F-012 の機能詳細が未記載**
  機能一覧に登録された 12 機能のうち詳細記述があるのは F-001, F-002, F-004, F-006 の 4 機能のみ。残る 8 機能の仕様が欠落しており、実装者が自己解釈するリスクが高い。全機能 ID に詳細セクションを記述すること。

- **[非機能要件の欠落] 同時表示レイヤー数・複数レイヤー時の性能目標が未定義**
  パフォーマンス要件は「GeoJSON 10MB / 3 秒」のみで、複数レイヤー同時表示時の性能目標とレイヤー数上限が不明。実装・テスト基準が定まらないため追加すること。

- **[非機能要件の欠落] エラーハンドリング・ユーザー通知仕様が不十分**
  F-002 備考に CORS エラー時の通知が触れられるのみで、ファイル破損・未対応フォーマット・CSV 列未検出など他エラーケースの UX 仕様が要件に記載されていない。非技術者向けシステムとして不可欠な観点であり、エラーハンドリング要件セクションを追加すること。

### 🟡 Warning（推奨対応）

- **[スコープの明確化不足] 「将来対応」の境界線が未定義**
  スコープ外（認証・DB 保存・サーバーサイド・モバイル）は記載されているが、Nice to Have 機能が「今回スコープ内」か「将来対応」かが不明確。「Future Scope」セクションを設けて境界を明示すること。

- **[機能漏れ] ベースマップ切り替え（Should Have）がユーザーストーリー・機能一覧に未登録**
  3.2 重要機能に記載されているにもかかわらず、対応する US・F-xxx が存在しない。設計・実装・テストの追跡が不可能なため、US と F-xxx を追加すること。

- **[機能漏れ] 印刷・スクリーンショット機能（Nice to Have）が機能一覧に未登録**
  3.3 に記載されているが、機能一覧（F-001〜F-012）に対応する機能 ID が存在しない。F-013 として追加すること。

- **[非機能要件の欠落] 運用・保守要件が未記載**
  デプロイ方式・ビルド手順・依存ライブラリ更新方針など運用・保守観点が全くない。「運用・保守要件」セクションを追加すること。

- **[非機能要件の抽象化] アクセシビリティ要件が「考慮」止まり**
  WCAG 準拠レベル・スクリーンリーダー対応の要否・色覚特性への配慮（コロプレス配色）が未定義。WCAG 2.1 AA 等の具体的指標を明記すること。

- **[勝手解釈の疑い] F-006「自然分類（Jenks）」が上位仕様にない分類方法を追加している可能性**
  機能詳細で「等間隔/自然分類」が記載されているが、3.1 の要件には「コロプレス・段階彩色」のみ。ヒアリングで合意されたか確認し、根拠を記載すること。未合意なら縮小またはステークホルダー確認を行うこと。

### 🟢 Info（参考情報・提案）

- **受け入れ条件の定量化**：US-004（レイヤー管理）・US-007（フィルタリング）・US-008（検索）に「3 秒以内」等の定量基準がなく、テスト判定基準が曖昧。

- **Shapefile ZIP 構成要件の明記**：ZIP 内に必要なファイル（.shp / .dbf / .shx / .prj 等）の構成とエラー仕様を F-001 に追記すること。

- **成功指標の計測方法の定義**：セクション 6 の指標は全て定性的で、評価タイミング・計測方法が未定義。ユーザビリティテストの要否も検討すること。

- **WMS/WFS 対応スコープの明確化**：F-002 備考に「WMS/WFS」が言及されているが、3.1 は「REST API」のみ。別プロトコルとして対応スコープを明示的に合意すること。

## 総評

ユーザーストーリー・機能一覧・非機能要件の骨格は整い、非技術者を主対象とした目的定義も明確である。しかし機能詳細が 12 機能中 8 機能で欠落し、トレーサビリティの仕組みも不在であることが最大の課題。機能一覧からベースマップ切り替えと印刷機能が漏れており、現状では設計・テスト工程に持ち込むには情報が不十分。早期の補完が必要。
