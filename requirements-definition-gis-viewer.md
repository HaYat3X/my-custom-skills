# 要件定義書: GIS汎用ビューワー

## 1. 背景・目的

業務担当者（非技術者）が地理空間データを手軽に可視化・分析できる環境が不足している。高価な専用GISソフトや技術知識なしに、ブラウザ上でGISデータを閲覧・操作できる汎用ビューワーを構築することで、データドリブンな業務判断を促進する。

## 2. ターゲットユーザー

| ユーザー種別 | 特徴・利用シーン |
|---|---|
| プライマリ | 業務担当者（非技術者）：GIS専門知識なしに地図データを確認・分析したい現場担当者 |
| セカンダリ | GISエンジニア・データ担当者：データを整備してビューワーに取り込む管理者側の役割 |

## 3. 機能要件

### 3.1 必須機能（Must Have）

- **地図表示**: OpenStreetMapをベースマップとしてインタラクティブな地図を表示（パン・ズーム操作）
- **ファイルアップロード**: GeoJSON / Shapefile / GeoTIFF・ラスター / CSV（緯度経度付き）をブラウザから読み込み
- **URL/API取得**: 外部URLまたはREST APIエンドポイントを指定してデータを取得・表示
- **レイヤー管理**: 複数レイヤーの追加・削除・表示/非表示切り替え・重ね合わせ順序変更
- **属性情報ポップアップ**: 地物をクリックした際に属性情報をポップアップ表示
- **スタイル設定**: 属性値に基づく色分け（コロプレス・段階彩色）の設定

### 3.2 重要機能（Should Have）

- **フィルタリング**: 属性値の条件指定で表示地物を絞り込み
- **検索**: 属性値のキーワード検索で地物をハイライト・ズーム
- **凡例表示**: スタイル設定に対応した凡例をマップ上に表示
- **ベースマップ切り替え**: OSM以外のタイル（衛星画像等）への切り替えオプション

### 3.3 あると良い機能（Nice to Have）

- **データエクスポート**: 表示中・フィルタ済みデータのGeoJSON/CSVエクスポート
- **計測ツール**: 距離・面積の計測
- **共有URL**: 現在の表示状態をURLパラメータで共有
- **印刷・スクリーンショット**: 地図の印刷用出力

## 4. 非機能要件

| 項目 | 要件 |
|---|---|
| パフォーマンス | GeoJSON 10MB程度のデータを3秒以内に表示できること |
| ブラウザ対応 | Chrome / Firefox / Edge の最新2バージョン |
| レスポンシブ | デスクトップ（1280px以上）を主対象。タブレットでも基本操作が可能なこと |
| セキュリティ | アップロードされたデータはクライアントサイドのみで処理（サーバー送信しない） |
| アクセシビリティ | UI操作にキーボードナビゲーションを考慮 |

## 5. 制約・前提条件

- **技術スタック**: Vue.js（フロントエンドのみ）、地図ライブラリはLeaflet.jsまたはMapLibre GL JSを推奨
- **データ処理**: ファイル読み込み・変換はすべてクライアントサイドで実施
- **ベースマップ**: OpenStreetMap タイルを基本とする（無料・商用利用可）
- **スコープ外**: ユーザー認証・データの永続化（DB保存）・サーバーサイド処理・モバイルアプリ

## 6. 成功指標

- GIS非専門の業務担当者がマニュアルなしにデータをアップロードして地図表示できる
- 複数形式のデータを1つのビューワーで扱えることで、ツールの乱立が解消される
- フィルタリング・色分けにより、データから業務的な洞察を得られる操作フローが完結する

## 機能詳細

### F-001: ファイルアップロード
- **概要**: ドラッグ&ドロップまたはファイル選択ダイアログでGISデータを読み込む
- **入力**: GeoJSON / Shapefile (.zip) / GeoTIFF / CSV（lat/lon列含む）
- **出力/結果**: 読み込まれたデータが新規レイヤーとして地図に追加される
- **備考**: ShapefileはZIP圧縮形式で受け付ける。CSVはlat/lon列を自動検出する

### F-002: URL/APIデータ取得
- **概要**: 入力フォームにURLを貼り付けてデータを取得・表示
- **入力**: GeoJSONを返すURL / WMS/WFSエンドポイント
- **出力/結果**: 取得したデータが新規レイヤーとして追加される
- **備考**: CORSエラーが発生する場合はユーザーへエラーメッセージを表示

### F-004: レイヤー管理パネル
- **概要**: サイドパネルで全レイヤーを一覧表示し操作する
- **入力**: ユーザーのUI操作（ドラッグ、チェックボックス、削除ボタン）
- **出力/結果**: 地図上のレイヤー表示がリアルタイムに反映される
- **備考**: レイヤー名はファイル名またはURL由来で自動設定し、編集可能にする

### F-006: スタイル設定（色分け）
- **概要**: 属性列と配色スキームを選択してコロプレス表示に切り替える
- **入力**: 対象属性列・分類方法（等間隔/自然分類）・配色テーマ
- **出力/結果**: 地物の塗りつぶし色が設定に応じて変更される
- **備考**: 数値列は段階彩色、文字列列はカテゴリ色分けを適用する
