# 要件定義: 汎用GISビューワー

作成日: 2026-04-03
ステータス: Draft
担当: Claude

---

## 1. 背景・目的

GeoJSON・シェープファイル・CSV・タイルマップなど多様なGISデータを、専用GISソフトなしにブラウザ上で手軽に閲覧・可視化できる汎用ビューワーを提供する。ファイルをドラッグ&ドロップするだけで即座に地図に表示でき、レイヤー管理やスタイル変更も直感的に操作できることを目指す。

## 2. スコープ

### やること（In Scope）

- GeoJSON / シェープファイル (.shp) / CSV（緯度経度付き）/ タイルマップ（XYZ・WMS）の読み込みと表示
- ファイルのドラッグ&ドロップによる即時読み込み
- レイヤーパネルでの表示/非表示切り替え
- 地物クリックによる属性情報のポップアップ表示
- レイヤー単位でのスタイル変更（色・透明度・線の太さ・ポイントサイズ）
- ベースマップの切り替え（OpenStreetMap / 衛星写真など）
- 地図の基本操作（ズーム・パン・フィット）

### やらないこと（Out of Scope）

- データの編集・保存・エクスポート（閲覧専用）
- サーバーサイドでのデータ管理・永続化
- ユーザー認証・権限管理
- 空間解析・ジオプロセシング機能
- モバイルアプリ（PWA対応は将来検討）

## 3. ユーザーストーリー

| # | ユーザー | したいこと | 理由 |
|---|---------|-----------|------|
| US-01 | GISデータ担当者 | GeoJSONファイルをD&Dで即座に地図表示したい | QGISなどのGISソフトを起動せずに素早く内容確認したいから |
| US-02 | 調査担当者 | シェープファイルをブラウザで開いて属性を確認したい | 現地確認・報告書作成時に手軽にデータを参照したいから |
| US-03 | データアナリスト | 緯度経度付きCSVを地図にプロットしたい | 位置データの分布を視覚的に把握したいから |
| US-04 | 複数データ担当者 | 複数レイヤーを重ねてON/OFFしながら比較したい | データ間の空間的な関係を確認したいから |
| US-05 | 一般ユーザー | レイヤーの色やサイズを変えて見やすくしたい | デフォルト表示では重なって区別しづらいから |

## 4. 機能要件

### ファイル読み込み

- FR-01: GeoJSONファイル（.geojson / .json）をドラッグ&ドロップで読み込めること
  - 受け入れ条件: ファイルをビューワーエリアにドロップすると地図にレイヤーとして追加されること
- FR-02: シェープファイル（.shp + .dbf / .zip）を読み込めること
  - 受け入れ条件: ZIPアーカイブまたは個別ファイルをドロップするとGeometryと属性が正しく表示されること
- FR-03: 緯度経度カラムを持つCSVファイルを読み込めること
  - 受け入れ条件: lat/lng または latitude/longitude カラムを自動検出してポイントとして表示されること
- FR-04: XYZ タイル URL / WMS URL をUIから入力して追加できること
  - 受け入れ条件: URL入力フォームからタイルレイヤーが地図に追加されること

### レイヤー管理

- FR-05: 読み込んだレイヤーの表示/非表示をトグルで切り替えられること
  - 受け入れ条件: チェックボックスのON/OFFで地図上のレイヤーが即座に表示・非表示になること
- FR-06: レイヤーの重ね順を変更できること
  - 受け入れ条件: レイヤーパネルでドラッグして順序を入れ替えると地図の描画順が変わること
- FR-07: レイヤーを削除できること
  - 受け入れ条件: 削除ボタンクリックでレイヤーが地図とパネルから除去されること

### 属性表示

- FR-08: 地物をクリックすると属性情報がポップアップ表示されること
  - 受け入れ条件: ポリゴン・ライン・ポイントいずれもクリック可能で、全属性キー/値がポップアップに表示されること

### スタイル変更

- FR-09: レイヤー単位で塗りつぶし色・線色・透明度を変更できること
  - 受け入れ条件: カラーピッカーで選択した色が地図に即時反映されること
- FR-10: ポイントレイヤーのサイズ・形状（円/四角）を変更できること
  - 受け入れ条件: スライダーでサイズを変更すると全ポイントに即座に反映されること

### ベースマップ

- FR-11: ベースマップを複数から選択できること
  - 受け入れ条件: OpenStreetMap・衛星写真・地理院タイル（淡色）の3種以上を切り替えられること

## 5. 非機能要件

| 種別 | 要件 | 優先度 |
|------|------|--------|
| パフォーマンス | 5MB以下のGeoJSONを3秒以内に描画完了すること | Must |
| 対応ブラウザ | Chrome・Firefox・Edge の最新版で動作すること | Must |
| レスポンシブ | 1280px以上のデスクトップ解像度を主対象とする | Should |
| アクセシビリティ | キーボードナビゲーションで主要操作が可能なこと | Could |
| オフライン動作 | タイルマップ不使用時はオフラインでも動作すること | Could |

## 6. 制約・前提条件

- フロントエンドのみ（サーバーレス）で動作する静的Webアプリとして構築する
- 技術スタック: Vue 3 + OpenLayers
- シェープファイルのパースには `shpjs` ライブラリを使用する
- 大規模データ（10万件超のフィーチャー）の最適化は将来課題とする
- セキュリティ: ファイルはクライアントサイドのみで処理し、サーバーには送信しない

## 7. 用語定義

| 用語 | 定義 |
|------|------|
| レイヤー | 地図上に重ねて表示するデータの一単位 |
| フィーチャー | 地図上の地物（ポイント・ライン・ポリゴン） |
| 属性 | フィーチャーに付随する表形式のデータ（名称・面積など） |
| ベースマップ | 背景として表示するタイル地図（OSMなど） |
| GeoJSON | GISデータを表現するJSON形式の標準フォーマット |
| シェープファイル | ESRIが定めたGISデータの標準フォーマット（.shp等） |
