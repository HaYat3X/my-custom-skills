# Notion Summarizer スキル

NotionページのURLを受け取り、ページ内容を取得して要約する。

## 前提条件

- Notion MCPが接続済みであること（`notion-fetch` ツールが利用可能）

## ワークフロー

### Step 1: URLの受け取りと確認

ユーザーからNotionページのURLを受け取る。URLが含まれていない場合はURLを尋ねる。

対応するURLフォーマット：

- `https://www.notion.so/...`
- `https://notion.so/...`
- ワークスペース付き: `https://www.notion.so/{workspace}/{page-id}`

### Step 2: コンテンツ取得

Notion MCPの `notion-fetch` ツールを使ってページ内容を取得する。

```
notion-fetch(url: <ユーザーから受け取ったURL>)
```

取得に失敗した場合：

- ページが非公開 or 権限なし → ユーザーに共有設定を確認するよう伝える
- URLフォーマット不正 → 正しいURLを再入力するよう促す

### Step 3: 要約の生成

取得したコンテンツを以下の方針で要約する：

**要約の基本方針**

- 日本語で出力する（元コンテンツが英語でも日本語で要約）
- ページの目的・主題を冒頭で一文で述べる
- 重要なポイントを箇条書きで列挙する
- ページの種類（タスクリスト、ドキュメント、メモ等）に応じて要約スタイルを調整する

**ページ種別ごとの要約スタイル**

| ページ種別           | スタイル                                             |
| -------------------- | ---------------------------------------------------- |
| タスク・TODO         | 未完了タスクを優先してリストアップ、期限があれば明示 |
| ドキュメント・仕様書 | 目的→背景→主要内容の順で構造化                       |
| 会議メモ             | 決定事項・アクションアイテムを抽出                   |
| 日記・メモ           | キーワードと主要トピックを抽出                       |
| データベース         | レコード数・主要カラム・注目エントリを報告           |

**要約の出力フォーマット**

```
## 📄 ページ概要
（ページの目的・主題を1〜2文で）

## 🔑 主要ポイント
- （ポイント1）
- （ポイント2）
- ...

## 📝 詳細メモ（必要な場合のみ）
（補足が必要な情報があれば）
```

コンテンツが少ない（500文字以下）場合はフォーマットを省略してシンプルに要約してよい。

### Step 4: 追加アクションの提案（任意）

要約後、ユーザーの文脈に応じて以下を提案してもよい：

- 「特定のセクションだけ詳しく知りたい場合は教えてください」
- ページ内にサブページがある場合、「サブページも確認しますか？」
